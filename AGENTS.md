## Workspace Instructions

- Treat existing campaign instance and character data in this repository as disposable test data unless the user explicitly says otherwise. Treat campaign modules as non-disposable. 
- When making or reviewing breaking changes that affect stored campaign or character data, safely ignore backward-compatibility concerns by default.
- Assume the database can be reset, dropped, or otherwise nuked unless the user explicitly instructs otherwise.
- If a task would destroy local data and the user has not said data must be preserved, prefer the simplest path instead of adding migration or preservation work.
- All gameplay and story creation should be AI-generated by default; local deterministic fallback dungeon-master logic should not exist and should be treated as a detriment to player experience.
- When debugging or fixing issues in the content generation pipeline, communicate findings and reasoning as high-quality handoff material for another agent that does not currently have codebase access, so outside-agent review and opinion consolidation are easy.
- Do not add regex-based natural-language parsing/classification heuristics when a router/model decision can provide the signal instead; treat regex attempts to catch or parse user language as a brittle last resort to avoid rather than a normal solution.
- Long-running commands like `pnpm -s tsc --noEmit` and `pnpm test` may crash WSL and interrupt the Codex session. If the user later says something like `pick up where we left off or continue working`, assume the previous session may have died mid-task or mid-verification. Resume by inspecting the current worktree and recent changes first (`git status`, `git diff`, relevant touched files), preserve partial progress, and continue from the last likely checkpoint instead of asking the user to restate the task from scratch.
